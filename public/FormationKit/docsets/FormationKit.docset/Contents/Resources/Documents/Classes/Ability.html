Ability

public class Ability

Every action, command, spell, buff, etc comes down to an Ability. This class mainly serves as a container for the data while other portions of the code make the magic happen. Let’s go over some terminology which shows the ways an Ability may interact with other portions of the code.

  • Ability: Base class which contains all information about the ability including name, description, cooldown, particles and more.
  • Behavior: Enum describing the different types of abilities offered.
  • Modifier: An actual change an ability may incur. For example, +10 HP or +20 Speed
  • AbilitiesComponenet: Entry point for processing an ability on a Squad. Will either run it immediately, or add to Queue via Command.
  • Command: An ability on its own will do nothing. A Command is a container for abilities that allows Squads to work with it. A squad will execute a command either instantly, or added to its QueueComponent.
  • CommandInstructions: Abilities have no context. CommandInstructions provide the context by specifying target, location, etc.

  • Action: A wrapper around an ability specificly for UI purposes. Actions will update icons when they are locked, cooling down, etc and give UI feedback.

    • Warning

    An Ability should never be modified outside of its construction. Abilities are loaded once and cached. Units with shared Abilities share the same pointer to the instance.

    Behaviors
    • Behavior

      Hard coded options for how an ability can behave. Used across the entire code base.

      • Move: Travel to valid lcoation. Targets self
      • Melee: Travel to engage range and attack. Targets enemy
      • Ranged: Attack immediately if target in range. Targets enemy
      • Instant: Immediately process ability when activated. Targets self
      • Spell: Combine animation and range with immediate action. Targets enemy
      • AoE: Instant ability, but is processed through physics body
      • Passive: Applied when unit is loaded and remains for lifetime of unit
      • Halt: System ability to stop attacking. Cleans up all references and stops a squads actions. Ranged only.
      public enum Behavior : String
    Specialty
    • Specialty

      Hard coded options for hero abilities – defines which special feature to use

      • Casting: Special animation / effects to run before using a spell. Stops squad if walking
      • Lunge: Melee attack where the hero jumps towards the target
      public enum Specialty : String
    Validation
    • Validation

      Ability can only be used if the Validation criteria is met.

      • Moving: Squad is moving.
      • NotMoving: Squad isn’t moving.
      • InCombat: Squad is in combat.
      • NotInCombat: Squad is not in combat.
      • Engaging: Walking towards enemy target. Melee only
      • EngagingTargetRange: Range of target that is beign engaged already. Different than combat range. Useful for abilities like Charge that can only be used when engaging, and within a certain distance
      public enum Validation : String
    • ValidationData

      Validation can get complex as this goes on. By storing each validation object in a struct, we can build out variable criteria like engaging, but within range of engaged target Abilities have command instructions, but those only include target. Sometimes the target may be a friendly unit, but the validation criteria may need checks against a different unit.

      public struct ValidationData
    Targeting
    • Targeting

      Used by UI and AI to determine what targets are valid

      • Myself: Only the unit with the ability
      • Friendly: Friendly units relative to unit casting.
      • Enemy: Enemy units relative to unit casting.
      • Anyone: Any squad alive.
      public enum Targeting : String
    Sound Effects
    • SoundMark

      Each ability will register different timing marks depending on .Behavior

      • Triggered: The ability was successfully triggered. Could be squad based sound like group of feet marching instead of individual footsteps on each unit.
      • Acknowledged: Play sound accepting the triggering of the ability. Example would be UI playing voice command
      • PreAnimation: Right before animation begins. Example would be casting a spell sound
      • AnimationTriggerPoint: Right when event occurs, like an arrow releasing from a bow
      • Success: Ability has finished. Arrow has reached target. Sword hits. Etc
      • Fail: Ability has completed but did not succeed. Arrow missed, etc.
      public enum SoundMark : String
