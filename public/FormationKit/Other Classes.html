  • Ability

    Every action, command, spell, buff, etc comes down to an Ability. This class mainly serves as a container for the data while other portions of the code make the magic happen. Let’s go over some terminology which shows the ways an Ability may interact with other portions of the code.

  • Ability: Base class which contains all information about the ability including name, description, cooldown, particles and more.
  • Behavior: Enum describing the different types of abilities offered.
  • Modifier: An actual change an ability may incur. For example, +10 HP or +20 Speed
  • AbilitiesComponenet: Entry point for processing an ability on a Squad. Will either run it immediately, or add to Queue via Command.
  • Command: An ability on its own will do nothing. A Command is a container for abilities that allows Squads to work with it. A squad will execute a command either instantly, or added to its QueueComponent.
  • CommandInstructions: Abilities have no context. CommandInstructions provide the context by specifying target, location, etc.

  • Action: A wrapper around an ability specificly for UI purposes. Actions will update icons when they are locked, cooling down, etc and give UI feedback.

    • Warning

    An Ability should never be modified outside of its construction. Abilities are loaded once and cached. Units with shared Abilities share the same pointer to the instance.

  • FKDefendingState

    The purpose of this state is to wait while being attacked by a Melee unit for the first time. When engaged, this state will trigger, which causes the defender to stop moving and wait. The attacker will seek, and once the attacker has reached the defender, the defender can then switch to a normal combat state. This is to allow things like charge, unit targeting, etc to work reliably. If a unit is already engaged in Melee combat, it would not enter this state should another attacker join the fight.

  • FKSharedAttributesComponent

    Example: Squad casting time, which is stored in abilities component. Flow is:

     - AttributesComponent.getvalueFor(.Attribute)

 - AttributesComponent stores a delegate, and calls delegate.getBaseValueFor(.Attribute)

     -  If none exsists, return nil. Same with setters (just ignore the set). If permannet, delegate.setPermenanentChangeFor(.Attribute, value)

     - Setter would also call delegate.attributeDidChange(.Attribute, currentValue) and custom logic can be run to update other parts of the code

     - Could also add attributeDidPermanentlyChange if the distinct is needed

 - The base value is taken, and then all modifiers will be searched to see if the value should be altered.

 - Final, altered value is returned.

 - Now, when abilities componenrt checks casting time, it will look like:

    if elapsedTime > (ability.cooldown * squad.attributesComponent.getValueFor(.CastingSpeed)) { // ready to cast again. }

    class FKSharedAttributesComponent : GKComponent

  • FriendlyFireRule

    The goal of this rule is to check if engaging would result in friendly fire. If so, lean heavily towards another target. Only try to fire if some condition like Beserk makes you, or if the friendly in combat is likely to lose, so firing into the crowd won’t hurt anymore. If no friendly fire, return 100. If friendly fire, 0. Or, scaled down, 0 to 0.05

    class FriendlyFireRule : FKFuzzyRule

  • LikelyToWinRule

    Goal of this rule is to attack targets you’re more likely to defeat. Simple version is to check total squad health against subject squads damaage and vice versa. If one is greater, that determines who should win. Can get more detailed to check modifiers, weaknesses, etc

    class LikelyToWinRule : FKFuzzyRule
  • ObstacleInterferenceRule

    The goal of this rule is to check for obstacles. If engaging results in movement around the environment, there is a small penalty. If moving through a friendly squad, a larher penalty. If moving thorugh an enemy sqiad, the largest penalty. Also, for ranged units, if they are in range and have line of sight, then this rule shouldn’t matter. Rewards: enemy in way = 0 clear path = 50 which return 0.05
