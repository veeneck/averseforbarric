FKAnimationComponent

class FKAnimationComponent : GKComponent

Handles running a requested animation. Other components will request starting and stopping animations. In theory, this could be expanded to prevent certain state changes (i.e.: cant go from walking to attacking), but that shouldn’t be required for this simulation.

  • animations

    Cached textures to use

    var animations: [FKAnimationComponent.AnimationState : FKAnimationComponent.Animation]
  • requestedAnimation

    Pending animation

    var requestedAnimation: FKAnimationComponent.AnimationState?
  • currentAnimation

    Handle on current animation for easy lookup

    var currentAnimation: FKAnimationComponent.AnimationState?
Processing Animations
  • resolveRequestedAnimation()

    Run an animation if it isn’t already running. Or, if .Idle is requested, stop animations.

    func resolveRequestedAnimation()
  • runAnimation(_:request:)

    Runs the animation of it exists in cache. Will run once or repeat based on .shouldRepeat. Updates handle on current animation.

    private func runAnimation(_ unit: FKUnitEntity, request: AnimationState)
Update Loop
  • update(deltaTime:)

    Constantly checking to see if an animation is requested. If so, put it in motion. The animation will run as a SKAction once or on a loop. Requesting .Idle will stop any animation.

    override func update(deltaTime seconds: TimeInterval)
Data Structure
  • Animation

    Defines everything need for the animation to run. In the future, could be expanded to allow different timing per frame and other advanced functionality.

    class Animation
Animation States
  • AnimationState

    Different keys used to classify animations. Important: Never set a ket an animation for Idle unless we create idle animations. For now, it resets back to base frame. Note: The string key is used to tag the SKAction

    • Idle
    • WalkFront
    • WalkBack
    • AttackFront
    • AttackBack
    • GetHit
    • Death
